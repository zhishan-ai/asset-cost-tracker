# SPEC

## 产品概述
面向移动端与桌面端的 Web 端资产管理小工具，以纯前端方式呈现，围绕资产录入、展示、计算与数据持久四大核心能力，界面强调圆角、柔和阴影、轻动效，并在空状态提供友好提示。

## 数据字段与校验
- `id`（唯一标识，自动生成，非空）。
- `name`（资产名称，必填，不能全空白，推荐限制字符数在 60 字以内）。
- `price`（价格，必填，正数，小数点后最多两位，输入时允许千位分隔符但存储为数字；用于计算总资产与日均）。
- `purchaseDate`（购买日期，必填，本地日期；禁止未来日期；至少以当日为上限，不可为空）。
- `status`（可选，枚举，如 `全部`/`使用中`/`收藏中`，用于后续筛选面板；默认 `全部`）。
- `icon`（可选，emoji 或图标路径，展示在资产卡片左侧）。

所有字段在用户交互时须实时校验，提示语需清晰（如“价格需大于 0 且最多两位小数”）。

## 核心计算公式（必须遵守）
以本地“今天”为基准，所有计算都采用原始数据，不透露缓存过时信息。

1. `daysHeld_i = max(1, diffDays(today, purchaseDate_i) + 1)`，即购买当天为第 1 天，今日也算在内。
2. `dailyCost_i = price_i / daysHeld_i`，内部保持高精度，展示时四舍五入保留两位小数。
3. `totalCost = sum(price_i)`。
4. `totalDailyAvg = sum(dailyCost_i)`。

## UI 结构与交互
1. **总体结构**：顶部大卡片展示“总资产（总金额）”与“总日均（总日均成本）”，下方按照资产列表展示每项资产卡片；卡片采用圆角与阴影，整体背景留白。
2. **资产卡片内容**：左侧展示 `icon`，中部 `name` 与 `status`（如有），右侧展示 `price`、`dailyCost`（¥/天）与 `daysHeld`（天）；每个数值应有 hint 标签。
3. **交互/状态**：
   - 卡片支持轻微 hover 高亮与微动效（例如 scale 1.02）。
   - 支持空状态文案：如“还没有资产，快去添加第一笔吧！”。
   - 提供“新增资产”按钮弹窗/抽屉输入必要字段，支持快速清空表单。
   - 自动按 status 筛选显示（全部/使用中/收藏中 可切换）。
4. **响应式**：
   - 移动端：一列布局，卡片充满宽度，间距适配。
   - 桌面：卡片可多栏，顶部 summary 卡片横向展示，列表下方可并排排列。
5. **动效**：卡片渐显、序列化 stagger Animation；中间 summary card 数值过渡动画（如抖动/淡入）。

## 数据持久与边界情况
- 使用 `localStorage` 持久化资产列表，保存版本号 `assetManager_v1`；遇到版本不符需自动迁移或重置（需保留迁移提示日志）。
- 初始化时读取 `localStorage`，若数据为空或结构错误进入“空状态”表现。
- 边界条件：价格为 `0` 不能保存；购买日期不能晚于今天；天数最低为 1，避免除以 0。

## 验收标准
- 页面加载后能读取并展示 `localStorage` 中的资产，空列表展示空状态文案。
- 资产新增/删除/编辑后数据及时写回 `localStorage`，刷新后保留。
- 所有数据卡片展示的 `daysHeld`、`dailyCost`、`totalCost`、`totalDailyAvg` 与公式一致。
- 页面在跨日后（根据当地 midnight timer）自动刷新计算，不需手动刷新。
- UI 在移动/桌面下均可正常展示，按钮/交互友好，动画表现不影响性能。
